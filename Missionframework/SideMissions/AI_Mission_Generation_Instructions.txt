# AI Mission Generation Instructions (_4–_99)

## Purpose

These instructions apply only to **community-created missions** in the `_4–_99` range.

* Missions `_1–_3` are reserved for core/internal use.
* Do NOT overwrite or duplicate `_1–_3`.

---

## 1. Mission Categories

Allowed mission categories:

* DisruptMission
* SupplyMission
* IntelMission
* MainOperation

Each mission must be placed in the correct category folder.

Example:

* SideMissions/modern/DisruptMission/
* SideMissions/modern/SupplyMission/

---

## 2. Theme / Content Pack Structure

Missions must be placed inside a valid theme pack:

Examples:

* modern
* gm
* vietnam
* ww2
* custom packs

Structure:
SideMissions/{theme}/DisruptMission/
SideMissions/{theme}/SupplyMission/
SideMissions/{theme}/IntelMission/
SideMissions/{theme}/MainOperation/
SideMissions/{theme}/compositions/

---

## 3. Mission ID Rules (IMPORTANT)

Before creating a mission, ALWAYS:

1. Scan the target folder
2. List existing mission IDs
3. Find used numbers
4. Select the next available ID between `_4–_99`

Example:

* If MissionSupply_4 exists → DO NOT create another _4
* Use the next free ID (e.g., _5)

---

## 4. Reserved Range

* `_1–_3` → core/internal missions
* `_4–_99` → community missions

Never modify `_1–_3`.

---

## 5. NO Hardcoded Classnames

Mission files must be **mod agnostic**.

DO NOT hardcode:

* Units
* Vehicles
* Props
* Objects
* Factions

❌ Example (NOT allowed):
"O_Soldier_F", "B_Truck_01_transport_F"

---

## 6. Unit / Vehicle Sources

### OPFOR

* Source: preset/opfor
* Scaled by: world tier

### BLUFOR

* Source: preset/blufor
* Scaled by: player base tier

### Civilians

* Source: preset/civ

### Rebels

* Source: preset/rebel
* Scaled by: rebel tier
* May depend on reputation

---

## 7. Compositions (REQUIRED)

All mission environments must use **composition keys**.

Examples:

* supply_cache_small
* intel_meeting_site
* roadblock_light
* field_hq_medium

DO NOT place objects manually.

---

## 8. Mission Data Standard

Each mission must define:

* missionId

* title

* description

* category

* difficulty

* allowedZoneTypes

* allowUndercover

* usesOpfor

* usesBluforSupport

* usesCivilians

* usesRebels

* compositionKey

* rewardSupplies

* rewardIntel

* rewardThreat

* rewardTier

* rewardThreatUndercover

---

## 9. Reward Rules

* Rewards are **fixed per mission**
* Rewards can vary between missions
* Do NOT randomize rewards

Undercover:

* Uses rewardThreatUndercover (usually 0)

---

## 10. Difficulty

Allowed:

* Easy
* Medium
* Hard

Difficulty affects:

* complexity
* resistance
* reward

NOT classnames.

---

## 11. Placement Rules

Missions must spawn in valid zones only.

Examples:

* town
* military
* factory
* capital
* rural
* roadside

---

## 12. Rebel / Reputation Rules

If a mission uses rebels:

* May require reputation
* May fail if reputation is too low
* May restrict access

---

## 13. Variety

Ensure variation in:

* category
* difficulty
* stealth vs loud
* civilian presence

Avoid repetition.

---

## 14. Safety Rules

Community missions must NOT:

* modify presets
* modify economy
* modify tier/threat systems
* modify core framework logic

---

## 15. Generation Workflow (MANDATORY)

1. Select theme
2. Select category
3. Scan existing missions
4. Find free ID (_4–_99)
5. Build mission data
6. Use NO classnames
7. Use compositions
8. Use presets/tier systems
9. Define fixed rewards
10. Save mission

---

## 16. Internal Scope

* `_1–_3` → internal development only
* `_4–_99` → community expansion

---

END OF INSTRUCTIONS
